The Challenge

Retro game art relies on sharp, aliased edges and limited color palettes. Displaying these on modern 4K monitors requires massive upscaling.

  • Standard methods (Bicubic) produce blurry, muddy results.
  • SOTA models (SRGAN, SwinIR) interpret sharp pixel boundaries as "noise," creating a destructive "oil painting" effect.

Dataset

We utilized the Kaggle Pixel Art Dataset (89,399 images). Unlike standard pipelines, we developed a geometric approach:

  • NN-Downsampling: LR-HR pairs created strictly via Nearest-Neighbor interpolation.
  • Preserves blocky geometry, teaching the model that LR-HR mapping is geometric, not smooth.

Live Demo

We integrated the model into a fully playable Mario engine. A real-time hotkey ('T') toggles between original NES graphics and our 4x ESRGAN assets to verify hit-box fidelity and visual clarity during gameplay.

Custom Loss Functions

We introduced two novel losses tailored to retro assets:

  • Edge-Aware Sharpness: Sobel-based penalty that forces hard, binary edges.
  • Palette-Constraint: Penalizes colors outside discrete 8-bit grids, preventing "color drifting."

Conclusion

  • Domain-specific adaptation + massive data scaling completely solves pixel-art super-resolution.
  • Custom Edge + Palette losses drastically outperform generic SOTA models.
  • Future work: Real-time browser extension for retro game emulators.
