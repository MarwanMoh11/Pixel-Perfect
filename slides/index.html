The Problem

Why Standard Upscaling Fails

  • Retro games run at 256×240 resolution — tiny on 4K monitors
  • Bicubic/Bilinear interpolation produces blurry, muddy images
  • SOTA SR models (SRGAN, SwinIR) are trained on photorealistic data
  • They interpret sharp pixel edges as "noise" and smooth them out
  • Result: an "oil painting" effect that destroys the artistic intent

What We Need

  • Sharp, binary edges — no anti-aliasing
  • Limited color palettes preserved
  • Grid-aligned pixel geometry respected
  • 4× resolution increase (32×32 → 128×128)
Hard Edges No Blur Pixel Fidelity

Dataset & Preprocessing

Kaggle Pixel Art Dataset

  • 89,399 total pixel-art images (sprites & objects)
  • All images are 16×16 resolution
  • 80/10/10 Train / Validation / Test split
  • 71,519 train · 8,939 val · 8,941 test
  • ~500 MB — feasible for Colab training

Nearest-Neighbor Downsampling

  • HR sprites resized to 128×128 via NN interpolation
  • LR counterparts created at 32×32 via NN downsample
  • Preserves blocky geometry — no smooth blur introduced

Dataset statistics: split distribution, size distribution, and dominant colors

Sample Preprocessing Pipeline

Pipeline Stages

  • Raw HR — Original sprite from dataset
  • NN Downsampled — 8×8 low-res via Nearest-Neighbor
  • Processed LR — Resized back for model input

Key Property

Unlike bicubic which introduces smooth gradients, NN downsampling maintains hard pixel boundaries — teaching the model that the LR-HR relationship is geometric, not smooth.

Key Design Choices

  • No Batch Normalization in Generator — prevents artifacts
  • Residual scaling (0.2) for stable deep training
  • Dense connections within each block (5 conv layers)
  • 8 RRDB blocks reduced from 23 — sufficient for pixel-art geometry

VGG-Style Discriminator

  • Processes 128×128 patches
  • 10 conv layers with BatchNorm + LeakyReLU
  • Progressive downsampling: 128→64→32→16→8→4
  • Binary classifier: Real vs. Generated

Proposed Updates

Update 1

Nearest-Neighbor Downsampling

Standard SR models train on bicubic-downsampled LR images (which are blurry). We create all training pairs using Nearest-Neighbor downsampling — preserving blocky pixel geometry and teaching the model that the LR→HR mapping is geometric, not smooth.

# Nearest-Neighbor preserves hard pixel edges hr_image = TF.resize(img, [128, 128], interpolation=InterpolationMode.NEAREST) lr_image = TF.resize(hr_image, [32, 32], interpolation=InterpolationMode.NEAREST)
Update 2

Edge-Aware Sharpness & Palette Loss (Milestone 2)

We combine a Sobel edge-penalty to remove "oily" blur, and a new Palette-Constraint Loss that snaps generated colors to a strict 8-bit/16-bit grid, solving the color drifting problem.

# Penalize smooth transitions (anti-aliasing) grad_x = conv2d(gray, sobel_x, padding=1) grad_y = conv2d(gray, sobel_y, padding=1) magnitude = sqrt(grad_x² + grad_y²) # Penalty peaks for "smooth transition" magnitudes penalty = relu(mag - 0.05) * relu(0.3 - mag)
ℒtotal = 1.0 · ℒL1 + 0.1 · ℒLPIPS + 0.5 · ℒEdge + 0.05 · ℒPalette + 0.005 · ℒAdv

Perceptual Quality — LPIPS

Why LPIPS Matters Most

  • PSNR favors blurry outputs (penalizes any deviation equally)
  • LPIPS measures perceptual similarity — how humans actually see quality
  • Our model: 0.0022 LPIPS
  • Baseline: 0.5082 LPIPS
  • That's a 230× improvement in perceptual quality
